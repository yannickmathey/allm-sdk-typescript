# Security policy

## Supported versions

Security fixes are provided for the latest `1.x` release. Older releases may be asked to
upgrade before a fix is issued.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability
reporting from the repository's **Security** tab and include:

- the affected SDK version and runtime;
- a minimal reproduction;
- the security impact;
- any suggested remediation.

Do not include live ALLM API keys, provider credentials, customer data, or other secrets in a
report. Revoke any credential that may already have been exposed.

Reports will be acknowledged as soon as practical. A disclosure timeline will be coordinated
after the issue has been reproduced and a remediation is available.
